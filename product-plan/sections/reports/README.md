# Reports Section

## Overview

The Reports section provides a read-only browsing interface for system-generated reports. Fleet owners and administrators can view, download, and share reports that are automatically generated by the LOTS247 platform.

Reports are organized by type using a tabbed interface:

- **All** -- Shows every report across all types
- **MIS** (Monthly Incident Summary) -- Monthly rollup of all incidents across the subscriber's fleet
- **ICR** (Incident Closure Report) -- Generated when a specific incident is closed, linked to the incident and vehicle
- **ISR** (Incident Summary Report) -- Generated 24 hours after a specific incident is created
- **MIS-CHALLAN** (Monthly Challan Summary) -- Monthly rollup of all challan activity across the fleet

## Features

- **Tabbed navigation** with per-tab report counts
- **Search** by period, vehicle registration, incident ID, or report type
- **Format filter** (PDF / Excel) via an expandable filter panel
- **Desktop table view** with sortable columns (Period, Report Type, Format, Generated, Actions)
- **Mobile card view** with responsive layout
- **PDF preview modal** showing report metadata, type-specific summary data, and download/share actions
- **Download** action to save reports as PDF
- **Share via WhatsApp** action
- **Pagination** with page numbers and prev/next navigation
- **Empty state** when no reports match the current filters

## Components

| Component | File | Description |
|-----------|------|-------------|
| `ReportsList` | `components/ReportsList.tsx` | Main component -- tabbed report browser with search, filters, table/card views, and preview modal |

## Data

- **Types**: `types.ts` -- `Report`, `ReportType`, `ReportTab`, `ReportsProps`
- **Sample data**: `sample-data.json` -- 11 sample reports across all 4 report types

## Props

The `ReportsList` component accepts data and callbacks via `ReportsProps`:

- `reports` -- Array of `Report` objects
- `activeTab` -- Optional controlled tab state
- `onTabChange` -- Callback when user switches tabs
- `onPreview` -- Callback when user previews a report
- `onDownload` -- Callback when user downloads a report
- `onShareEmail` -- Callback for email sharing
- `onShareWhatsApp` -- Callback for WhatsApp sharing
- `onSearch` -- Callback when user types in the search bar

## Usage

```tsx
import { ReportsList } from './components'
import sampleData from './sample-data.json'

<ReportsList
  reports={sampleData.reports}
  onDownload={(id) => console.log('Download', id)}
  onShareWhatsApp={(id) => console.log('Share', id)}
/>
```
